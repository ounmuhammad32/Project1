What is Git?

Git is a revision control system which arose out of the Linux kernel development community, which had a very specialized need. As thousands of developers spread worldwide, on many different time zones, working on a very complicated project, it was important to be able to coordinate all that work in a rational way and not lose track of what was going on. So, Git was designed for that purpose and it has grown to be used by millions of other projects today, and is the underpinning of most open source software work that's being used on this planet today.

What is GitHub?

GitHub is a code hosting platform for version control and collaboration. It lets you and others work together on projects from anywhere using repositories, branches, commits, and pull requests. It is commonly used to host open-source projects. As of November 2021, GitHub reports having over 73 million developers and more than 200 million repositories (including at least 28 million public repositories). It is the largest source code host as of November 2021.

Main Git/Github Vocabulary

  • Repository - the . git/ folder inside a project, it tracks all changes made to files in your project, building a history over time. It is your whole project (directories and files) that you clone on your computer
  • Branch - a version of your repository, or in other words, an independent line of development, it allows you to develop features, fix bugs, or safely experiment with new ideas in a contained area of your repository
  • Merge - a common practice for developers using version control systems. Whether branches are created for testing, bug fixes, or other reasons, merging commits changes to another location. To be more specific, merging takes the contents of a source branch and integrates them with a target branch. In this process, only the target branch is changed. The source branch history remains the same
  • Commit - a commit is something that you create, typically by editing files in the working tree, adding them to the index, and then saying git commit
  • Tag - Tags are ref's that point to specific points in Git history. Tagging is generally used to capture a point in history that is used for a marked version release (i.e. v1. 0.1). A tag is like a branch that doesn't change
  • Pull - used to fetch and download content from a remote repository and immediately update the local repository to match that content
  • Push - sends the recent commit history from your local repository up to remote repository
